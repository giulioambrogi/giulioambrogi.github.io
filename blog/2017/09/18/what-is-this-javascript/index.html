What is this in JavaScript

JavaScript may look an easy language for anyone with an Object Oriented Programming background, however some of its features are being totally misunderstood. This is because it was built according to the function pattern, rather than the OOP pattern.

One of the most misunderstood keyword is this. What is it anyway?

In a common Object Oriented Programming language such as C#, this keyword is a reference to the current instance of the class. See the example below.

Pretty intuitive so far.
In JavaScript the story is highly different: this is a reference to the context where a function have been called at runtime. I know, this definition my look blurry at this stage but it will be much clearer as we go through some examples.

As you can see, even if both person.getName and getName reference the same function, the value of this is not the same, because it depends on the context in which getName() is being called: the first time it’s being called on top of person context, on the other hand - the second time - it’s being called on the global context.

So remember, in JavaScript, the value of this is a matter of context - AKA where the function is being called from.

Different contexts

Now let’s examine how this behaves when a function is being called in different contexts: the global scope and the local scope.

When called on the global scope this points the global object, that is window in browsers or globals in Node.
