What Is This in JavaScript

What is this in JavaScript?

JavaScript may look an easy language for anyone with a OOP backgrund, by the way some of its features are being totally misunderstood due to the fact that JavaScript was not built using OOP pattern, but the function pattern.

One of the most misunderstood keyword is this. What is it anyway?

In a common Object Oriented Programming language such as C#, this keyword is a reference to the current instance of the class. See the example below.

In JS the story is slightly different: this is a reference to the context where a function have been called at runtime.
I know, this definition my look blurry at this stage, it will be much clearer as we go through some examples.

As you can see, even if both person.getName() and getName() reference the same function, the value of this is not the same, because it’s based on the context in which getName() is being called.

So remember, in JavaScript, the value of this is a matter of context - where the function is being called from.

Different contexts

Now let’s examine how this behaves when being called in different contexts.

When called on the global scope this points the global object, that is window in browsers or globals in Node.
